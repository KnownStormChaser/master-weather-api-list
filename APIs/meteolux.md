### MeteoLux

Free, public weather API from MeteoLux — the official meteorological service of Luxembourg, operated by the Administration de la navigation aérienne (ANA) — providing forecasts, current conditions, official warnings (vigilances), and real-time airport observations for Luxembourg. Data is released into the public domain (CC0).

- **Based in:** Findel, Luxembourg (Luxembourg Airport; MeteoLux is the meteorological service of the Administration de la navigation aérienne, ANA)
- **Access model:** No key required for the public data endpoints — public HTTPS REST API with interactive OpenAPI/Swagger documentation, queried by coordinates. A client-credentials token endpoint (`POST /auth/client/token`) exists for application-specific access (e.g. the MeteoLux app), but the open weather/observation data is served without it. GET (HEAD on some endpoints); a test WebSocket endpoint is also exposed
- **Pricing model:** Free (open public-service data; no fees and no paid tiers)
- **Coverage / data types:** current conditions, 5-day forecasts (daily plus sub-daily 6-hourly steps), hourly wind forecast, and official weather warnings/vigilances (multilingual — English, French, German, and Luxembourgish); forecast parameters include weather condition, temperature, precipitation amount, wind, UV index, and sun & moon / sunshine duration. A separate **HVD** branch (EU High Value Datasets directive) serves real-time 1-minute observations from Luxembourg-Findel Airport, with a stations list and observation metadata. Reverse geocoding (coordinates → nearest location) is included. The published response schemas also model radar, satellite, road status, and climatology objects, though the headline products are forecast / current / warnings / Findel observations
- **Free tier:** the entire API is free; no published per-key quota (the public data endpoints use no key)
- **Paid tiers:** none — there is no commercial or higher-service tier
- **Rate limits:** none published (fair use)
- **Output formats:** JSON (REST API documented via an OpenAPI 3.1 / Swagger schema at `/api/v1/docs`); imagery endpoints stream image files
- **Geographic strength:** Luxembourg only — a small, single-country domain centred on the Grand Duchy, with real-time observations from the Luxembourg-Findel station
- **Notable limitations:** Luxembourg-only coverage (single-country); the forecast parameter set is fairly narrow (no humidity, dew point, pressure, cloud cover, or precipitation probability surfaced in the forecast); sub-daily forecasts come in 6-hour steps rather than true hourly (a separate hourly wind forecast is the exception); historical depth is limited — the HVD observations are real-time/recent rather than a deep archive; lookup is by coordinates with reverse geocoding but no place-name search; the interactive Swagger page is the primary reference (no separate prose developer guide). Data is released under the Creative Commons CC0 1.0 Universal (Public Domain Dedication) — provided "as is", no warranty, and no attribution legally required (crediting MeteoLux remains good practice), with the ANA giving no guarantee of continued availability
- **Built on:** MeteoLux's own forecaster-analysed forecasts and the Luxembourg-Findel observation station; the underlying NWP model(s) are not enumerated in the API documentation
- **Docs:** https://metapi.ana.lu/api/v1/docs
- **Last verified:** 2026-06
