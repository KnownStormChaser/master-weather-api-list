### Environment and Climate Change Canada (MSC GeoMet)

Free, open geospatial platform from Environment and Climate Change Canada (ECCC) and the Meteorological Service of Canada (MSC) — Canada's official source for weather forecasts, alerts, observations, radar, climate, hydrometric, and air-quality data, served through standards-based OGC web services and a REST OGC API. Access is anonymous and key-less.

- **Based in:** Gatineau, Quebec, Canada (Environment and Climate Change Canada HQ); the Meteorological Service of Canada's Canadian Meteorological Centre (CMC), which runs the operational forecast models, is in Dorval, Quebec
- **Access model:** No key required — MSC GeoMet access is anonymous and free; public HTTPS endpoints are queried directly with no signup. GET only. (The companion MSC Datamart raw-file server is a separate bulk-download option.)
- **Pricing model:** Free (open government data; no fees and no paid tiers — ECCC does run separate "cost-recovered services" for specialized/custom needs, quoted independently)
- **Coverage / data types:** Canada-wide weather forecasts and public alerts/warnings, surface weather observations (SWOB), radar and satellite imagery, numerical model output (global GDPS/GEPS and regional RDPS/HRDPS), climate data (daily/hourly observations, normals, LTCE records), hydrometric water-level/flow station data, Air Quality Health Index (AQHI observations & forecasts), and meteorological text bulletins — delivered via WMS map layers, WCS coverages, and the OGC API (Features + Coverages); a STAC catalog is experimental
- **Free tier:** the entire platform is free and key-less; the OGC API Features endpoint caps responses at 10,000 features per query (page through results with `limit`/`offset` or the response's `next` link)
- **Paid tiers:** none on the open platform; separate cost-recovered services exist for specialized data needs (custom / contact)
- **Rate limits:** no published per-user quota (fair-use); services are operational 24/7 with best-effort user support during normal business hours; the 10,000-feature response cap effectively bounds single-query size
- **Output formats:** features — GeoJSON (JSON), HTML, CSV; coverages — CoverageJSON, GRIB, GeoTIFF, NetCDF; WMS image tiles. Server-side filtering via `bbox`, `datetime`, property queries, and CQL2 (comparison, logical, spatial predicates)
- **Geographic strength:** Canada — the authoritative national source for forecasts, alerts, observations, climate, and hydrology. Some served products are global NWP model grids (GDPS/GEPS cover the whole globe), but the curated forecast/alert/observation products are Canadian
- **Notable limitations:** primarily a geospatial/GIS-oriented platform (OGC WMS/WCS/OGC-API) rather than a plain "point → forecast JSON" REST API — it's best used with software that supports geospatial web services, so there's more of a learning curve than a simple forecast endpoint; the public forecast/alert/observation products are Canada-only (worldwide coverage is limited to raw model-output grids); OGC API Features responses are capped at 10,000 features per request; data is licensed under ECCC's open-data / Open Government Licence – Canada terms, which require attribution
- **Built on:** MSC/CMC's own numerical weather prediction systems — the Global (GDPS) and Global Ensemble (GEPS), and the Regional/High-Resolution (RDPS/HRDPS) systems, all part of the GEM model family — plus the national radar and satellite networks, surface and hydrometric station networks, and ECCC's climate archives
